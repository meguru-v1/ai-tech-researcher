# Skill: 日次レポート生成 (report-daily)

## 目的
本日収集された技術情報を集約し、読みやすい日次レポート（Markdown形式）を生成する。また、各情報のソースを評価し、採用実績を記録する。

## 手順
1. **データの取得**:
   - `collected_data` から本日収集されたレコードを全件取得する。
   - `sources` テーブルの情報も結合し、どのキーワード由来のデータか分かるようにする。

2. **レポートの要約**:
   - LLMを用いて、収集された情報をカテゴリ分け（例：AIツール、LLM動向、開発フレームワーク）し、重要なトピックを要約する。
   - 抽出された各記事について、タイトル、URL、短い要約を含むレポートを作成する。

3. **採用実績の記録**:
   - レポートに採用（掲載）した記事のソースIDを特定する。
   - `adoption_logs` テーブルに `is_adopted = 1` として記録する。
   - 採用されなかった（ノイズと判断された）記事のソースIDは `is_adopted = 0` として記録する。

4. **レポートの保存**:
   - 生成されたMarkdownを `reports/daily/{YYYY-MM-DD}.md` に保存する。
   - `reports` テーブルにレコードを挿入する。

5. **完了報告**:
   - 生成されたレポートのパスを報告して終了する。
